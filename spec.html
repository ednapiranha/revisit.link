An API playground.

Specifications

  1. The following is the only REST call the service needs to provide to users:
    • POST /service where JSON is posted to you in the following format: 
          {
      "content": {
          "type: "image/jpeg",
          "data": {DATA URI}
      },
      "meta": {
          "audio": {
              "type": null,
              "data": null
          }
      }
  }
      and returns the following format: 
          {
      "content": {
          "type: "image/jpeg",
          "data": {DATA URI}
      },
      "meta": {
          "audio": {
              "type": null,
              "data": null
          }
      }
  }
  2. All content from a service can only be accessed by that service. Nothing is stored on the central hub except for script links to other service APIs per token.
  3. Any content passed through the API cannot exceed 1 MB per request per media item (content and audio count as two items).
  4. All code for the service must be open source.
  5. If any of the services violate the rules listed above, they will be removed from the central hub.
  6. Once you are ready to include your service to the central hub, add your domain and send a pull request to this file.
